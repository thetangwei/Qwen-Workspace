# GeoSphere AR - 互动地球仪应用

## 项目概述

**项目名称**: GeoSphere AR (暂定名)

**产品愿景**: 打造一款结合物理实体与数字信息的沉浸式地理探索工具。通过手机摄像头，将静态的地球仪转化为动态的、可交互的3D数字星球，提供直观的手势控制和丰富的地理信息展示。

**目标用户**:
- K-12 学生及教育工作者
- 地理爱好者/旅行者
- 科技体验追求者

## 核心功能需求

### 3.1 AR 初始化与锚定

- **FR-01 平面/物体检测**: 启动App后，引导用户扫描桌面或特定的物理地球仪图案。
- **FR-02 持久化锚点**: 一旦识别成功，建立世界坐标系锚点，确保虚拟地球仪与物理位置严格对齐。
- **FR-03 重定位机制**: 若追踪丢失，提供视觉引导帮助用户重新对准。

### 3.2 手势交互系统 (Gesture Interaction System)

- **FR-04 单指/手掌拨动旋转**:
  - 算法识别手部关键点（指尖或手掌中心）
  - 计算手部在屏幕空间的移动向量 Δx, Δy
  - 将向量映射为地球仪的经纬度旋转增量
  - 优化：加入死区过滤（防止微小抖动导致旋转）和灵敏度调节

- **FR-05 双指捏合缩放**:
  - 识别双手食指拇指距离或单手双指距离
  - 控制相机与地球仪中心的距离（Zoom In/Out）
  - 设置最小/最大缩放级别，防止穿模或过远

- **FR-06 视角跟随（Parallax Effect）**:
  - 利用手机IMU（陀螺仪/加速度计）数据
  - 当用户围绕地球仪移动手机时，虚拟相机视角随之改变，产生视差效果，增强3D立体感

### 3.3 视觉渲染与效果

- **FR-07 高保真地球渲染**:
  - 支持日间/夜间模式切换
  - 云层动态流动效果
  - 大气层散射光晕

- **FR-08 实时光照估计**:
  - 获取环境光探针数据，实时调整地球仪漫反射和高光
  - 在地球仪下方生成基于平面的实时阴影

- **FR-09 遮挡渲染**:
  - 利用深度图，实现手部遮挡虚拟地球仪，或地球仪遮挡背景的真实物体

### 3.4 信息交互层 (Info Layer)

- **FR-10 POI 标记**: 在主要城市、山脉、海洋位置悬浮3D图标。
- **FR-11 点击详情**: 点击虚拟标记，弹出半透明UI卡片，显示名称、人口、简介等。
- **FR-12 路径绘制**: 可选功能，两点之间绘制大圆航线。

## 算法与逻辑设计

### 4.1 手势到旋转的映射算法

```csharp
// 伪代码示例
public void MapHandGestureToRotation(Vector2 handDelta)
{
    // 应用死区过滤
    if (handDelta.magnitude < deadZoneThreshold)
        return;
    
    // 计算旋转增量
    float rotationX = handDelta.y * rotationSensitivity;
    float rotationY = -handDelta.x * rotationSensitivity;
    
    // 应用旋转到地球仪
    earthSphere.Rotate(rotationX, rotationY, 0);
}
```

### 4.2 坐标系转换

- **Screen Space -> World Space**: 将手部2D屏幕坐标通过射线投射（Raycast）或与虚拟球体相交计算，转化为3D空间中的旋转轴和角度。
- **Camera Pose Integration**: 将手机的 Transform.position 和 Transform.rotation 与AR摄像机的位姿解耦，允许用户在不移动物理手机的情况下，通过滑动屏幕微调视角。

## UI/UX 设计规范

- **极简主义**: AR场景中UI应尽量少，避免遮挡视野。
- **引导式教学**: 首次启动时，通过半透明动画演示"如何拨动"、"如何缩放"。
- **反馈机制**:
  - 视觉：手势识别成功时，手部周围出现微弱光效。
  - 触觉：缩放至极限时震动。
- **无障碍设计**: 支持语音播报地名，支持高对比度模式。

## 风险与挑战

| 风险点 | 影响程度 | 应对策略 |
|--------|----------|----------|
| 手部追踪抖动 | 高 | 使用卡尔曼滤波（Kalman Filter）平滑手部坐标数据；增加死区判断 |
| 光照不足导致追踪丢失 | 中 | 提示用户增加环境光；提供手动重置锚点按钮 |
| 低端机性能瓶颈 | 高 | 提供"性能模式"（降低贴图分辨率、关闭阴影/遮挡）；动态降级渲染质量 |
| 隐私问题 | 中 | 明确声明摄像头数据仅在本地处理，不上传云端；符合GDPR/CCPA规范 |

## 💡 给开发者的额外提示

### 关于"不同角度看到不同面"
这其实是AR的基本特性。只要你的虚拟地球仪是固定在AR世界坐标系中的，当用户拿着手机绕着桌子走时，Unity/Unreal的相机自然会渲染出地球仪的背面。你不需要为此写特殊算法，只需要确保地球仪的锚点（Anchor）足够稳定，不要随着手机移动而漂移。重点在于SLAM（即时定位与地图构建）的稳定性。

### 手势库选择
如果追求极致性能且主要面向iOS，直接用 ARKit 5+ 的 Hand Tracking API是最优解，无需引入第三方重型库。如果是Android或跨平台，MediaPipe 是目前最成熟的选择，但需要注意其在Unity中的集成包大小和CPU占用。

### 物理地球仪的辅助
如果希望体验更好，可以设计一个配套的"marker"（比如一个特殊的底座或贴纸），贴在物理地球仪底部。这样AR识别会更快速、更稳定，比纯靠识别球形物体要容易得多（球体缺乏特征点，难以进行稳定的视觉SLAM追踪）。

## 快速开始

### 前置要求
- Unity 2022 LTS 或更高版本
- AR Foundation 5.x
- MediaPipe Unity Plugin
- iOS: Xcode 14+ / Android: Android Studio

### 安装步骤
1. 克隆或下载本项目到本地
2. 使用Unity Hub打开项目
3. 等待依赖包自动安装
4. 打开 `Assets/Scenes/MainScene.unity`
5. 连接设备并运行

### 构建配置
- **iOS**: 最低支持 iOS 14.0，需要启用ARKit支持
- **Android**: 最低支持 Android 8.0 (API 26)，需要启用ARCore支持
